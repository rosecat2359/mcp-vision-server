# MCP Hub 用户使用教程

MCP Hub 是一个 MCP Server 管理平台，帮助你集中注册、配置、监控远程 MCP Server，加密托管 API Key，并生成标准客户端接入配置。

## 一、核心概念

| 概念 | 说明 |
|------|------|
| **服务器（MCP Server）** | 一个可被 AI 客户端调用的工具服务实例，通过 SSE 或 stdio 传输 |
| **API 密钥** | 访问 MCP Server 所需的鉴权凭证（如 Anthropic、OpenAI 的 API Key），加密存储 |
| **连接向导** | 引导你选服务器、选密钥、测试连接、生成 Claude Desktop 配置 |
| **工作区** | 注册时创建的隔离空间，你的服务器和密钥都属于当前工作区 |

## 二、首次注册

1. 访问平台地址（如 `https://mcp.apifw.cn:8443`）
2. 点击「开始使用」→ 注册
3. 填写工作区名称、邮箱、密码（至少 8 位）
4. 注册成功后自动登录，进入仪表盘

> 第一个注册的账号即为工作区管理员（Admin 角色）。

## 三、添加 MCP Server

1. 左侧导航点击「服务器」→ 右上角「添加服务器」
2. 填写表单：

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 自定义服务名 | `agnes-vision` |
| 传输模式 | SSE（HTTP 长连接）或 stdio（SSH 代理） | SSE |
| 端点 | 服务地址 | `https://api.example.com/sse` |
| 鉴权方式 | Bearer Token / mTLS / 无鉴权 | Bearer Token |
| API Key | 鉴权所需密钥（选无鉴权则不填） | `sk-...` |
| 标签 | 逗号分隔的分类标签 | `production, vision` |

3. 点击「创建服务器」

创建后服务器列表会显示卡片，包含状态指示（在线/离线/错误）、传输模式、鉴权方式、标签和端点。

## 四、管理服务器

点击列表中的服务器卡片进入详情页：

- **查看信息**：传输模式、鉴权方式、端点、创建时间、最近 Ping 时间、标签
- **立即检测**：手动触发 Ping，检查服务器是否可达
- **最近日志**：该服务器最近的连接事件
- **删除**：移除该服务器（不可撤销）

服务器状态说明：

| 状态 | 含义 |
|------|------|
| 在线 | 最近一次 Ping 成功 |
| 离线 | 从未 Ping 通或长时间无响应 |
| 错误 | Ping 返回错误 |

## 五、管理 API 密钥

左侧导航「API 密钥」用于集中托管各 Provider 的 API Key。

### 添加密钥

1. 点击「添加密钥」
2. 选择 Provider（Anthropic / OpenAI / OpenRouter / Groq / 自定义）
3. 填写标签（便于区分，默认用 Provider 名）
4. 粘贴 API Key
5. 保存

> 密钥使用 AES-256-GCM 加密存储，列表中只显示脱敏预览（如 `sk-abc...wxyz`）。

### 密钥操作

- **查看**：临时显示明文（关闭窗口后从内存清除，可一键复制）
- **测试**：验证密钥是否有效（返回有效/无效状态）
- **删除**：永久移除

## 六、连接向导（生成客户端配置）

用于生成 AI 客户端（如 Claude Desktop）接入 MCP Server 的标准配置。

1. 左侧导航「连接向导」
2. **第 1 步**：选择要接入的 MCP Server
3. **第 2 步**：选择 API Key（可选，服务器需要鉴权时选）
4. **第 3 步**：点击「开始测试」验证连接是否通
   - 成功：显示延迟，可继续生成配置
   - 失败：显示错误信息，返回检查服务器/密钥
5. **第 4 步**：生成 Claude Desktop 配置 JSON，点击「复制」粘贴到客户端配置文件

### Claude Desktop 配置文件位置

- macOS：`~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows：`%APPDATA%\Claude\claude_desktop_config.json`

将生成的 JSON 合并到该文件后重启 Claude Desktop 即可。

## 七、查看日志

左侧导航「日志」显示所有服务器的连接事件：

| 字段 | 说明 |
|------|------|
| 时间 | 事件发生时间 |
| 服务器 | 服务器 ID 前 8 位 |
| 事件 | 已连接 / 已断开 / 错误 |
| 消息 | 事件详情 |

## 八、设置

「设置」页显示当前账户和工作区信息：工作区名称、邮箱、角色、注册时间。

角色权限：

| 角色 | 权限 |
|------|------|
| 管理员（Admin） | 全部操作，含用户管理 |
| 操作员（Operator） | 管理服务器和密钥 |
| 访客（Viewer） | 只读查看 |

## 九、常见问题

**Q：添加服务器后状态一直是「离线」？**
A：检查端点是否正确、服务器是否对外可达、鉴权信息是否匹配。进入详情页点「立即检测」看具体错误。

**Q：API 密钥测试显示「无效」？**
A：确认密钥未过期、Provider 选择正确、密钥有对应服务的访问权限。

**Q：连接向导测试失败？**
A：常见原因：服务器端点不通、鉴权失败、传输模式选错。先在服务器详情页单独 Ping 确认服务器本身可达。

**Q：查看密钥明文后丢失？**
A：明文仅在查看窗口显示，关闭即清除。密钥本身仍加密存储在数据库，可随时再次查看。

**Q：忘记密码？**
A：联系工作区管理员重置，或参考运维手册在数据库层面处理。

## 十、快速上手流程

```
注册账号
   ↓
添加 API 密钥（若服务器需要鉴权）
   ↓
添加 MCP Server
   ↓
连接向导：选服务器 → 选密钥 → 测试 → 生成配置
   ↓
复制配置到 Claude Desktop → 重启客户端
   ↓
开始使用
```
